# BATTLE INTERACTION SPEC — 幻獣TGC（動き仕様）

Date: 2026-06-21
Scope: Vertical Slice（召喚・移動・攻撃・本陣攻撃・勝敗）の動きと数値。
Note: 数値は **[採用案]**。playtestで上書き可。すべての成否・ダメージは `src/core/**` の結果に従い、本Specは「結果の見せ方」のみを定義する。すべての演出は `AnimationQueue` に直列投入し、1演出ずつ解決する。

---

## 0. 原則

- 演出はロジックを変えない。`rules.*` が「不可」を返したら、演出は元位置へ戻すだけ（音・揺れ無し）。
- 1アクション＝1キュー項目。重なり順は core のイベント順（v2 §14-2: 能動側→相手）に一致させる。
- タイミングは ms または s で明記。イージングは既定 `Cubic.easeOut`（特記時のみ変更）。

## 1. レイアウト基準（※現行バグの是正を含む）

横画面を主とする。盤は中央、手札は盤に被らない下部トレイに収める。**すべての寸法はビューポート基準でスケールする**（固定pxで組まない）。

| 要素 | 仕様 [採用案] | 備考 |
|---|---|---|
| 盤サイズ | `min(vw, vh) * 0.78` の正方を画面中央に配置（縦は上方バイアスで手札トレイ分を空ける） | tokenは `LAYOUT.boardFraction` |
| 手札トレイ高さ | `vh * 0.26`、画面下端に固定。盤の描画域と**重ねない** | safe-area(下) を差し引く |
| カード幅 | `clamp(76px, vw*0.085, 132px)`、比率 5:7 | 現行の `vw*0.12` は横画面で過大→縮小 |
| 扇 半径 | **`handTrayHeight * 1.9`（固定520pxを廃止）** | 現行の固定`FAN_RADIUS=520`が見切れの主因 |
| 扇 角度 | 中央0°から両端 ±14°（〜7枚）。枚数で線形補間 | 現行±18°は端が立ちすぎ |
| カード重なり | 隣接40%（枚数増で詰める。最小可視幅 `cardWidth*0.45`） | |
| ピボットY | `screenBottom + radius - cardHeight`（**カード全高が必ずトレイ内に収まる**位置） | 見切れ防止の確定式 |

> 是正の要点: `HandView` の `FAN_RADIUS` と `pivotY` をビューポート連動に変更し、カード下端がトレイ内に必ず入るようにする。`CARD_WIDTH_FRACTION` を縮小。

## 2. タイルハイライト

- **召喚可能マス**: 縁 `accent #3DA9FC`、内側塗り α0.12、パルス周期 0.4s（α 0.12⇄0.22）。
- **移動可能マス**: 同 `accent`、塗り無しの枠ハイライトのみ。
- **攻撃可能マス（敵ユニット/本陣）**: 縁 `danger #E5484D`、塗り α0.14、パルスやや速め 0.32s。
- 解除は 0.1s でフェードアウト。
- 本陣マス（自陣最奥中央2マス）は常時、薄い `hq #E8C870` の隅マーカーを表示（§Visual参照）。

## 3. 召喚（手札→盤）

1. ドラッグ開始: カードを `scale 1.06`・影濃度+、他の手札を `α0.6` に沈める（0.1s）。
2. ドラッグ中: 合法マスを §2 のパルスで発光。指/カーソル直下のマスを `scale 1.04` で強調。
3. 合法マスでドロップ → **召喚成功**:
   - カードを縮小落下（`scale 1.0→0.0`・対象マス中心へ移動）0.25s `Back.easeIn`。
   - 着地で板に **ユニット出現**: `scale 0.6→1.0` 0.18s + リング状の着地パルス（`accent`, α0.3→0, 0.3s）。
   - 手札から当該カード消去、`HandView` 再レイアウト 0.15s。
4. 非合法でドロップ → **失敗**: カードを元スロットへ `0.18s easeOut` で戻す。揺れ・音は無し。合否判定は `rules.summon` に委譲。

## 4. ユニット移動

1. 選択: ユニットに `selected #FFFFFF` リング（太さ2px）。移動可能マスを §2 で発光。
2. 移動先タップ → ユニットが経路を滑る: 1マスあたり 0.12s `Sine.easeInOut`、最大 0.3s でクランプ（長距離は等速寄り）。
3. 到着でリング消灯、行動済みフラグを薄いグレーアウト（`α0.85`）で表現。
4. 移動のみ/移動後攻撃は core の許可に従う(v2 §12)。攻撃後の移動は不可。

## 5. ユニット攻撃（相互ダメージ）

1. 攻撃元選択→対象タップ。
2. **ランジ**: 攻撃側が対象方向へ `0.12s` 突き、戻り `0.10s`。最大移動はタイル半分。
3. **ヒット**: 接触の瞬間に **ヒットストップ 0.06s**（全体を一時停止）。
4. **画面シェイク**: 通常 `3px` / 本陣攻撃や致死は `6px`、減衰 0.18s。
5. **ダメージポップ**: 被弾側の上に数値、`danger` 色、`y -18px` 上昇しながら 0.5s でフェード。回復は `health #9FE6B0`。
6. **相互ダメージ**: 攻撃側・防御側が同時に core 算出値を受ける。表示順は被攻撃側→攻撃側のポップを 60ms ずらす。
7. **退場**: HP≤0 のユニットを `scale→0.9 + α→0` 0.22s で消す。装備も同時に除去（v2 §18）。

## 6. 本陣攻撃（ライフ）

- 攻撃範囲に相手本陣マスが入り、かつ本陣マスに敵ユニットが居ない場合のみ可（居れば守備駒、v2 §10）。
- ランジ→本陣マスに `danger` フラッシュ 0.12s →相手ライフ数値が減少（HUDで §7）。
- **反撃なし**。シェイクは `6px`。

## 7. HUD 反映

- ライフ変化: 数値をロール（増減を 0.3s で補間）、減少時に赤フラッシュ、回復時に緑。
- マナ: 開始フェーズで最大+1・全回復を、ピップ（点）の点灯アニメ 0.2s で表現。
- ターン切替: 「あなたの番／敵の番」をセンターに 0.6s バナーで提示後フェード。

## 8. カードタップ拡大

- タップで中央に拡大（`scale→約vw*0.22幅` 0.15s）＋背面暗幕 α0.6。背後の操作はブロック。
- 外側タップ/再タップで閉じる 0.12s。ドラッグ開始時は拡大を発火させない（閾値: 8px移動で拡大キャンセル→ドラッグへ）。

## 9. 勝敗

- ライフ0検出（core `winloss`）→ 全入力ロック → 0.4s 後に Result 画面へ。勝/負で配色（`accent`/`muted`）。
- 「メニューへ」で Menu に戻る。1試合の一周（メニュー→対戦→決着→メニュー）が成立すること。

## 10. 音フック（実装は後回し・フックのみ）

`sfx(name)` を各イベントに仕込む: `summon` `move` `hit` `hqHit` `death` `turn` `win` `lose`。VS段階では no-op で可。
