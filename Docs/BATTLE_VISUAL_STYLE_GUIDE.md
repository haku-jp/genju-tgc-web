# BATTLE VISUAL STYLE GUIDE — 幻獣TGC（見た目基準）

Date: 2026-06-21
目的: 「安っぽさ」を構造的に防ぐ。色・寸法は `src/view/theme.ts` のトークンを単一の真実とし、本書はその意味と運用を定義する。**末尾追記禁止・該当箇所を修正**（legacy-webの重ね塗り失敗の再発防止）。

---

## 0. 2026-06-21 ライブレビュー所見（是正対象）

現行デプロイ（GitHub Pages）を実機描画して確認した安っぽさの主因。本書はこれらの是正を含む。

1. **手札の扇が過大で盤に被り、下端で見切れ** → INTERACTION_SPEC §1 で是正（半径・カード幅をビューポート連動に）。
2. **横画面で盤が上寄せ・左右デッドスペース** → 盤を中央配置（§2）。
3. **HUDがデバッグ素テキスト** → パネル＋アイコン化（§5）。
4. **タイル/カードがフラットで奥行きゼロ、本陣マーク無し、3系統フレーム未反映** → §2〜§4。
5. **モノトーンで雰囲気が無い** → §1 の役割色を陰影とともに使う。

## 1. カラーパレット（`theme.ts` ミラー）

| 用途 | トークン | 値 |
|---|---|---|
| 深部背景 | `bgDeep` | `#0B0E14` |
| パネル | `panel` | `#141A24` |
| 主要文字 | `ink` | `#E8E0D0` |
| 補助文字 | `muted` | `#8B97A6` |
| 選択/移動/マナ | `accent` | `#3DA9FC` |
| 攻撃/ダメージ | `danger` | `#E5484D` |
| タイル明/暗/枠 | `tileLight`/`tileDark`/`tileBorder` | `#1B2331`/`#141A24`/`#2A3344` |
| 自軍ユニット | `playerUnit` | `#2F6F4F` |
| 敵軍ユニット | `enemyUnit` | `#7A2330` |
| 本陣 | `hq` | `#E8C870` |

運用: 役割色は意味で固定（accent=自分/操作可、danger=敵/攻撃、hq=本陣/金）。**新色を足す時は必ず `theme.ts` に追加**し、画面側でリテラル色を書かない。

## 2. 盤面

- 構図: **横画面主**。盤は `min(vw,vh)*0.78` の正方を画面中央（手札トレイ分だけ上方バイアス）。
- **浅い俯瞰**: 真俯瞰でなく、奥行きを感じる軽いタイル陰影。各タイルに上端ハイライト1px(`tileBorder`明)＋下端の落ち影で“厚み”を出す。市松配色（`tileLight`/`tileDark`）で盤の升目を視認しやすく。
- **本陣（盤外ヒーロー方式・採用）**: 現実装では本陣は盤上マスを持たず、各プレイヤーの盤外ヒーロー（HUDのライフ＝本陣体力）として扱う。盤の最奥端（自陣＝下端／敵陣＝上端）の盤外マージンに `hq #E8C870` の本陣ライン（横バー＋「自本陣／敵本陣」ラベル）を置き、方向の目印にする。盤マス上にはマーカーを置かない（存在しない当たり判定を匂わせないため）。`本陣を攻撃` はこのヒーローを対象とする。
- 自陣2列（召喚可能域）は、平常時にごく薄い `accent` の地（α0.05）で“自分の領域”を示す。
- タイルは表計算に見せない: 枠だけの長方形を避け、塗り＋陰影＋わずかな角丸(2px)で物体感を出す。

## 3. ユニット（盤上）

- トークンサイズ: タイルの `unitFraction 0.82`。
- 形: 角丸矩形のフレーム。自軍=`playerUnit`縁、敵軍=`enemyUnit`縁、地は`panel`。
- 表示: 上に名前（`ink`・最小13px）、下に `ATK / HP` を大きく（ATK=`ink`、HP=被ダメ時 `danger`寄りに）。
- 状態表現: 行動済み=α0.85グレー寄り、召喚酔い=左下に小さな砂時計アイコン、選択中=`selected #FFFFFF`リング。
- ダメージ既受: HPが初期未満なら HP 数字を `danger` 系に。

## 4. カード（手札・拡大）

- 比率 **5:7**（width:height）。寸法は INTERACTION_SPEC §1。
- **3系統フレーム**（種別で枠の形・帯色を変え一目で識別）:
  - 幻獣: 上部にATK/HP枠を持つ戦闘用フレーム。帯 `playerUnit` 系。
  - 魔術: 一回性を示す尖った/流れる帯。帯 `accent` 系。
  - 装備: 取付を示すリング/フック意匠。帯 `hq` 系。
- **レアリティ縁色**（枠の外周色）: 幻(I)=虹がかった金白 / 伝説(L)=`hq`金 / 希少(R)=`accent`青 / 標準(S)=`muted`銀。略号 I/L/R/S をカード隅に小さく表記（v2 §6）。
- 情報配置: 左上=コスト（`accent`の円地に白数字）、中央=アート枠（VSはプレースホルダ）、下帯=名前、幻獣のみ右下に `ATK/HP`。
- 可読性: テキスト最小 12px。コスト数字は最大級に。アートが無い間も**フレームの質感（縁・帯・内枠の段差）**で“カードらしさ”を担保する。

## 5. HUD

- 素テキスト禁止。左上に **自分パネル**、右上に **敵パネル**（`panel`地・角丸・内側1pxハイライト）。
- 各パネル: ❤アイコン+ライフ（大）、◆マナ ピップ（現在/最大を点で）、手番中はパネル縁が `accent`/`danger` で発光。
- ターン表示はセンターバナー（INTERACTION_SPEC §7）。`End Turn`/`本陣を攻撃` は右下、`accent`の実体ボタン（押下で0.05s凹み）。

## 6. タイポグラフィ

- 既定 `system-ui, sans-serif`（`FONT_FAMILY`）。見出し/数値は太字。
- 最小サイズ: カード本文12px / ユニット名13px / ライフ等のHUD数値は20px以上。
- 数値（ATK/HP/コスト/ライフ）は等幅感のある字面で“ゲーム数値”に見せる。

## 7. レイアウト共通

- safe-area（特に下端・ノッチ）を尊重。手札トレイは safe-area 内に。
- 横画面を主、縦画面は盤を上・手札を下に積む退避レイアウト（盤を `min(vw,vh)` 基準で縮小）。

## 8. OpenDuelyst（CC0）の使い方

- **借りてよい**: CC0なのでアセット・配色研究・盤/カードの“見せ方”の参考。プレースホルダにそのまま使用可（収益化も可）。
- **コピーしない**: 幻獣TGC独自の世界観・カード枠アイデンティティ（3系統フレーム、共生圏トーン）。Duelystのコードは旧スタックで写経元にしない（Unity Coreが写経元）。借用素材は `public/ASSETS.md` に出所を記録。

## 9. アンチパターン（やってはいけない）

- 画面側に色リテラルをハードコード（必ず `theme.ts`）。
- スタイルの末尾追記による上書き（該当箇所を直す）。
- 枠だけのフラットなタイル/カード（陰影・段差・帯で物体感を出す）。
- ロジック値と表示値の不一致（表示は core の結果のみ）。
